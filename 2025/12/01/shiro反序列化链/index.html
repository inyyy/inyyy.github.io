Fastjson反序列化

Fastjson 简介

Fastjson 是 Alibaba 开发的 Java 语言编写的高性能 JSON 库，用于将数据在 JSON 和 Java Object 之间互相转换。

提供两个主要接口来分别实现序列化和反序列化操作。

JSON.toJSONString 将 Java 对象转换为 json 对象，序列化的过程。

JSON.parseObject/JSON.parse 将 json 对象重新变回 Java 对象；反序列化的过程

所以可以简单的把 json 理解成是一个字符串。

Fastjson提供两个接口：

JSON.toJSONString:将对象序列化为json

JSON.parseObject/parse:将json转化为java对象

对于JSON.parseObject如果它反序列化的json字符串对应的对象在序列化时指顶了输出类型就需要设置它的第二个属性，否则无法进行反序列化

JSON.parse和JSON.parseObject的区别

JSON.parseObject返回的是JSONobject，而JSON.parse返回的是实际类型的对象，两者都可以将json字符串转化为java对象，JSON.parseObject的本质就是利用json.parse进行反序列化，但是json.parseObject会进行额外的一步操作，就是JSON.toJSON(obj),json.parse在反序列化时会识别并调用对象的setter方法和特定条件的getter方法，由于json.parseObject多了一步JSON.toJSON(obj)操作，会调用对象的所有setter方法和getter方法，在进行完JSON.toJSON操作后会将这个序列化的结果暂时放在内存中，并不是最终的输出结果，后续检查无误会立可进行反序列化，所以对于漏洞的触发来说JSON.parseJSON更容易触发漏洞，因为它会调用所以属性的setter和getter方法，即使属性不在json键值对中

漏洞demo

由于Object是所有类的父类，所以当我们把JSON.parseObject的属性设置为Object.class时理论上就可以反序列化出任意类，根据前面的理解，JSON.parseObject会触发对应的类的所有setter方法和getter方法以及构造器，所以当这三者中存在恶意类或者恶意代码块时就可以触发rce

根据前面了解到的应该能触发getter方法，但是实际上并没有触发getage方法，后续了解了一下Fastjson的反序列化机制，JSON.parseObject并不会触发所有的getter方法和setter方法，它好像和JSON.parse一样只有 json字符串里面的键值对才回去触发setter方法，

关键就是返回值类型要满足返回值类型继承自Collection或Map或AtomicBoolean或AtomicInteger或AtomicLong

我这里用一个返回值map测试一下

成功调用getp方法，里面有触发命令执行的代码块，但是经过我的测试我发现Fastjson1.2.24版本在反序列化时只会去调用json字符串中有的字段值，即使setter方法和getter方法符合Fastjson的反序列化调用条件也不会触发，和我之前网上了解到的不一样，难道是Fastjson版本不同，另外带上参数 Feature.SupportNonPublicField可以对私有属性进行调用

jdbcRowSetImpl链

1：jndi+RMI

在jdbcRowSetImpl这个类里面并没有触发恶意逻辑的setter方法和getter方法，也就是说并没有完整的gadget链，所以jdbcRowSetImpl链实际上使用的reference类，根据exp最后是利用setDataSourceName方法

setAutoCommit也是一个关键的方法，它是触发jndirmi的开关，当参数为true时，就会利用connect方法调用lookup方法，实际这个类就相当于一个RMI客户端

这个connect方法最为关键，在这里可以看出这个jdbcRowSetImpl类可以充当一个jndiRMI攻击的客户端，这里利用py起一个简单的服务器

最开始一直失败，找了下报错是默认使用本地最高版本的jdk进行编译了，导致版本和jdk8不匹配

但是其中我发现在JdbcRowSetImpl类和他的父类里面没有dataSourceName和autoCommit字段，了解了发现Fastjson里面实质上不是依赖字段，而是依赖属性

在一个私有的方法defineTransletClasses方法里面被调用，我们还是不能直接进行调用，先看一下怎么new一个，再找权限转换的地方。第一个参数就是一个类加载器，看第二个，调用了_tfactory字段的一个方法，这个字段是一个TransformerFactoryImpl类，找到对应的方法

该方法返回一个Map，这里只需要反射_tfactory字段为一个类，到这里找到了defineTransletClasses方法进行调用

类的加载有一个重要的阶段：loadclass –findclass–defineclass loadclass是类加载的入口，findclass负责找到对应的字节码/class文件，defineclass负责将字节码转化为class对象，后续初始化阶段是最重要的，初始化发生在实例化阶段，因为初始化阶段会执行静态代码块，我们构造一个恶意字节码，在其中加上恶意静态代码块，在TemplatesImpl类中有一个方法实现了初始化，实例化了一个类

该方法调用了defineTranslletClasses，并且利用字节码进行了初始化，之前学的cc3字节码链到这里就能触发，但是在fastjson里面这个getter方法并不会被触发，它不满足Fastjson触发getter方法的条件，Fastjson要求getter方法返回的类型是集合，Map，long，Integer，Boolean，根据已知的EXP是找调用了这个方法的地方
