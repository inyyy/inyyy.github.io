spel表达式学习

spel是在spring3中引入的spring表达式语言

spel的功能：

1：使用Bean的id来引用Bean

2：可调用方法和访问对象的属性

3：可对值进行算数，关系和逻辑运算

4：可使用正则表达式进行匹配

5：可进行集合操作

一：使用Bean的id来引用Bean

javaBean：javaBean就是一个遵守特定规范的普通java类，用于封装数据的容器，用来取数据和存数据，需要满足的条件如下：

1：这个类必须是public修饰

2：有一个无参的公共构造方法

3：所有字段都是私有的

4：提供public的getter/setter方法

常见场景：

  • 封装前端传过来的参数
  • 封装数据库查询出来的一行记录
  • 在方法之间传递数据
  • 在框架（Spring、SpringBoot、Shiro、MyBatis）中被自动处理

使用Bean的id来引用Bean是spring框架灵活实现依赖注入的基石，它允许你在配置中直接通过id获取和引用spring容器中已经存在的Bean，spel使用作为界定符，当spring容器在创建和配置Bean时遇到#{}时就会将其中的内容作为spel表达式来解析和执行，要引用一个Bean，直接在#{}里面天上目标bean的id即可

使用Bean的id来引用Bean做常见的场景就是将一个类作为另一个的属性，这里比如一个类为person，一个类为car，spel最简单的是可以对一个类进行赋值，这里我在person类里面写了一个属性来学习一下

三：支持运算

直接在#{}界定符里面拼接运算就行，比如#{1+1}，结果会变成2

还支持逻辑运算and（&&），or（||），not（！）
还支持关系运算

支持 >、<、==、>=、<=。

在 XML 中，< 和 > 符号是特殊字符（用于标签），所以必须使用转义字符或替代符号：

< 写成 html编码或者 lt

> 写成 html编码形式或者 gt

== 可以写成 eq

五：类类型表达式

在spel表达式中，使用T(Type)运算符会调用类的作用域和方法，可以直接操作类，包括那些危险的类，这个Type必须是类的全限定名，它操作的是类本身，并不是一个类的实例化对象，所以它只能访问那些静态方法和属性，在反序列化中常用到的恶意类Runtime在这里也可以用来执行命令

可以利用spel表达式的T(Type)来执行命令

六：spel表达式在@Value表达式中的用法

spel表达式在@Value注解中的使用主要是用来读取配置，调用Bean方法和设置默认值，在@Value注解中使用spel表达式和在xml配置中不一样，在xml中使用界定符#{}，在@Value注解中是使用spring占位符$,使用”#{${}}”双符号嵌套，$占位符只能进行简单的读取配置操作，然后替换字符串，不能像#一样进行运算，逻辑操作，$可以嵌套在#中，但是反过来不行，当两者嵌套就是先读取配置在进行spel判断

1：读取配置信息和设置默认值

// 1. 直接读取 application.properties 中的 key
@Value(“${app.name}”)
private String appName;

// 2. 读取并设置默认值（如果找不到 key，则使用 ‘Unknown’）
@Value(“${app.version:Unknown}”)
private String version;

2：注入其他Bean或者调用Bean的方法

实例代码：

3：使用系统环境变量或系统属性

// 读取 JVM 启动参数 -Duser.home
@Value(“#{systemProperties[‘user.home’]}”)
private String userHome;

// 读取操作系统环境变量 (例如 PATH)
@Value(“#{systemEnvironment[‘PATH’]}”)
private String path;

4：使用T()运算符，实现对类的操作，访问静态方法和属性

// 调用 Math.random()
@Value(“#{T(java.lang.Math).random() * 100.0}”)
private double randomNumber;

// 使用静态常量
@Value(“#{T(java.lang.Integer).MAX_VALUE}”)
private int maxInt;

5：#和$的区别

@Value(“${}”) 读取配置并且替换字符串

@Value(“#{}”) 进行spel表达式判断

@Value(“#{${}}”) 先读取配置值再进行spel判断

@Value(“key:default”) ，冒号后根默认值

七：Expression中的用法

Expression是spel表达式的核心，Expression首先要依赖解析器，生成一个解析器，解析器将spel表达式翻译成spring框架可以看懂的语言，Expression得到这个翻译好厚的内容，后续负责取值和赋值工作，主要方法是getValue(),setValue()

1:使用getValue()方法进行取值

ExpressionParser parser = new SpelExpressionParser();
// 解析表达式，生成 Expression 对象（即“魔法卷轴”）
Expression exp = parser.parseExpression(“‘Hello’.concat(‘ World’)”);

// 调用 getValue() 触发执行，拿到结果
String result = exp.getValue(String.class);
System.out.println(result); // 输出: Hello World

先生成一个表达式解析器，Expression得到spel表达式逻辑解析后的内容，后续可从中取值

2：使用setValue()方法赋值

User user = new User(“张三”);
EvaluationContext context = new StandardEvaluationContext(user);

ExpressionParser parser = new SpelExpressionParser();
// 解析一个“赋值表达式”
Expression exp = parser.parseExpression(“name = ‘李四’”);

// 调用 setValue 或 getValue 都能触发这个赋值动作
exp.getValue(context);

System.out.println(user.getName()); // 原对象的 name 已经被动态改成了 “李四”

3：还可以直接访问Bean的属性

User user = new User(“张三”, 25);
// 把 user 设为根对象
EvaluationContext context = new StandardEvaluationContext(user);
ExpressionParser parser = new SpelExpressionParser();

// 表达式可以直接写 “name”，SpEL 会自动去根对象（user）里找
Object name = parser.parseExpression(“name”).getValue(context);
System.out.println(name); // 输出: 张三

总结Expression处理的步骤

1：先创建一个解析器，使用ExpressionParser接口表示解析器，提供SpelExpressionParser来实现

2：通过ExpressionParser接口的parseExpression方法解析表达式并得到一个Expression（”可执行的对象“），程序会将parseExpression方法里面的参数当作spel表达式处理，不需要定界符

3：声明一个上下文对象，表明后续的方法是对谁进行操作

4：取值和赋值，通过getValue()和setValue()方法
